❓ 什么是代码压缩（Minify）？

代码压缩（Minification）是指在不改变代码功能的前提下，去除源代码中的空白字符、换行符、注释、冗余分号等非必要内容，从而减小文件体积、加快网页加载速度的过程。这是前端性能优化的基础手段之一，广泛应用于生产环境的 HTML、CSS 和 JavaScript 文件。

CSS Minify去除空格、换行、CSS注释、最后一个分号、冗余0值20% - 40%样式表、主题CSS、组件库
JS Minify去除空格、换行、JS注释、console语句（可选）30% - 60%脚本文件、SDK、Webpack输出

🚀 为什么需要代码压缩？

  • 减少带宽消耗：压缩后的文件体积可减少 20%-60%，大幅降低服务器带宽成本。
  • 加速页面加载：更小的文件 = 更快的下载速度 = 更好的用户体验。Google 研究表明页面加载时间每增加1秒，转化率下降7%。
  • 提升SEO排名：Google PageSpeed Insights 将文件压缩作为核心性能指标，直接影响搜索排名。
  • 改善 Core Web Vitals：LCP（最大内容绘制）和 FID（首次输入延迟）都受益于更小的资源文件。
  • 保护源代码：压缩后的代码可读性大幅降低，一定程度上增加逆向工程难度。

💡 使用小贴士

  • 保留源文件：压缩前务必备份原始代码，压缩后的代码难以阅读和调试。
  • 搭配 Source Map：生产环境使用压缩代码 + Source Map，方便线上调试。
  • 自动化流程：项目中使用 Webpack/Vite 等构建工具自动压缩，本工具适合快速测试和临时需求。
  • 注意HTML中的 pre/code 标签：HTML压缩会保留 <pre> 和 <textarea> 内的空白和换行。
  • JS压缩注意事项：本工具不会修改变量名（不做混淆），仅去除空白和注释，保证功能完全不变。

📋 常见问题

Q: 压缩后的代码还能正常运行吗？
A: 能。本工具只去除不影响代码执行的空白、换行和注释，功能完全不变。

Q: 支持批量压缩吗？
A: 当前版本支持单文件压缩。如需批量处理，建议使用构建工具（Webpack/Vite）或 CLI 工具（Terser/cssnano/html-minifier）。

Q: 代码会上传到服务器吗？
A: 不会。所有压缩操作完全在您的浏览器本地执行，代码不会离开您的电脑。

Q: JS 压缩会缩变量名吗？
A: 不会。本工具是「美化级压缩」（去除空白注释），不做变量名混淆。如需 Uglify 级别的混淆压缩，请使用 Terser。
