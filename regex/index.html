正则表达式完全指南

正则表达式（Regular Expression，简称Regex）是一种用于匹配、搜索和操作文本的强大工具。无论是数据验证、文本提取还是批量替换，正则表达式都能以极少的代码完成复杂的文本处理任务。

什么是正则表达式？

正则表达式由普通字符（如字母a-z）和特殊字符（称为"元字符"）组成。它描述了一种字符串匹配模式，可以用来检查一个字符串是否含有某种子串、替换匹配的子串、或从某个串中取出符合某个条件的子串。

常用元字符速查表

符号含义示例
.匹配除换行符外的任意字符a.b 匹配 "aab"
^匹配字符串开头^Hello 匹配以Hello开头
$匹配字符串结尾end$ 匹配以end结尾
*匹配前一个字符0次或多次ab*c 匹配 "ac","abc","abbc"
+匹配前一个字符1次或多次ab+c 匹配 "abc","abbc"
?匹配前一个字符0次或1次ab?c 匹配 "ac","abc"
{n}匹配前一个字符恰好n次a{3} 匹配 "aaa"
{n,}匹配前一个字符至少n次a{2,} 匹配 "aa","aaa"...
{n,m}匹配n到m次a{2,4} 匹配 "aa","aaa","aaaa"
[abc]匹配a、b或c中任意一个[aeiou] 匹配任意元音
[^abc]匹配除a、b、c外的字符[^0-9] 匹配非数字
\d匹配数字 [0-9]\d+ 匹配一个或多个数字
\w匹配字母数字下划线 [a-zA-Z0-9_]\w+ 匹配一个单词
\s匹配空白字符\s+ 匹配一个或多个空白
\b匹配单词边界\bcat\b 匹配独立单词"cat"
(x|y)匹配x或y(http|https) 匹配两种协议
(...)捕获分组(\d{3})-(\d{4})
(?:...)非捕获分组(?:abc)+ 重复但不捕获

标志 (Flags) 说明

标志名称说明
g全局匹配查找所有匹配项，而非在第一个匹配后停止
i忽略大小写匹配时不区分大小写
m多行模式使 ^ 和 $ 匹配每行的开头和结尾，而非整个字符串
sdotAll使 . 也能匹配换行符 \n
uUnicode启用完整的 Unicode 支持，正确处理四字节字符

正则表达式常用场景

正则表达式在前端开发、后端验证、数据处理等领域有广泛应用：

  • 表单验证：邮箱格式、手机号码、身份证号、密码强度等前端验证
  • 数据提取：从HTML/XML中提取信息、解析日志文件、匹配URL参数
  • 文本替换：批量替换模板变量、格式化文本、代码重构
  • 爬虫开发：匹配和提取目标网页内容
  • 语法高亮：编辑器中的关键词着色
  • 路由匹配：Web框架中的URL路由规则

JavaScript 正则表达式方法

  • test() — 测试字符串是否匹配模式，返回 true/false
  • exec() — 执行匹配，返回匹配结果数组（含分组）
  • match() — 字符串方法，返回所有匹配结果
  • matchAll() — 返回所有匹配的迭代器（含分组）
  • replace() — 替换匹配的文本
  • search() — 返回第一个匹配的位置索引
  • split() — 使用正则分隔字符串为数组

常见陷阱与最佳实践

  • 贪婪 vs 懒惰：.* 是贪婪匹配（尽可能多），.*? 是懒惰匹配（尽可能少）
  • 回溯爆炸：嵌套的量词可能导致性能问题，避免 (a+)+ 这类模式
  • Unicode 陷阱：中文字符范围用 [\u4e00-\u9fa5] 而非 \w
  • 转义特殊字符：在正则中匹配 . * + ? [ ] ( ) { } ^ $ | \ 时需加 \ 转义
  • 善用在线测试工具：复杂正则建议先在测试器中验证，避免生产环境意外
