📖 什么是URL编码？

URL编码（百分号编码 / Percent-Encoding）是将URL中的特殊字符转换为 %XX 格式的过程。由于URL只能使用ASCII字符集中的部分字符，任何不在安全字符集中的字符（如中文、空格、特殊符号）都必须进行编码才能正确传输。

为什么需要URL编码？

  • 浏览器兼容性 — 各浏览器对未编码URL的处理方式不同，编码后可确保跨浏览器一致
  • 数据完整性 — 防止特殊字符（&、=、#等）被误解析为URL参数分隔符
  • 安全防护 — 防止XSS攻击和URL注入，对用户输入进行安全编码
  • API开发 — REST API中query参数和path参数的正确传输
  • SEO优化 — 搜索引擎能正确解析编码后的URL

💡 使用场景

1. 前端开发 — 在AJAX请求中传递包含中文或特殊字符的查询参数时使用 encodeURIComponent。

const query = encodeURIComponent('你好世界'); fetch('/api/search?q=' + query);

2. 表单提交 — 在表单数据通过URL传输时，对每个字段值进行编码。

const name = encodeURIComponent('张三'); const email = encodeURIComponent('zhang@example.com'); const url = '/submit?name=' + name + '&email=' + email;

3. URL解码调试 — 收到编码后的URL参数时，使用 decodeURIComponent 还原原始数据。

const raw = '%E4%BD%A0%E5%A5%BD%E4%B8%96%E7%95%8C'; const decoded = decodeURIComponent(raw); // 你好世界

4. Cookie/Header编码 — HTTP头部和Cookie值中不能包含非ASCII字符，需要URL编码。

❓ 常见问题

Q: encodeURIComponent和encodeURI有什么区别？
A: encodeURIComponent 用于编码URL的参数值（编码更多字符，包括 / ? & =等），encodeURI 用于编码完整URL（保留URL结构字符）。一般场景下用 encodeURIComponent 更安全。

Q: 为什么空格有时编码为%20，有时为+？
A: encodeURIComponent 将空格编码为 %20（RFC 3986标准）。+ 号是 application/x-www-form-urlencoded 格式的历史约定。推荐统一使用 %20。
