Cron表达式完全指南 — 从入门到精通

Cron表达式是Linux/Unix系统中用于定义定时任务的标准格式。无论你是DevOps工程师、后端开发者还是运维人员，掌握Cron表达式都能让自动化任务管理事半功倍。本工具提供可视化Cron表达式生成和实时预览功能，无需记忆复杂语法。

什么是Cron表达式？

Cron表达式由5个字段组成，每个字段用空格分隔，分别代表：分钟、小时、日期、月份、星期。一个完整的Cron表达式例如 0 9 * * 1-5 表示"每周一到周五的上午9点执行"。

Cron表达式字段详解

字段允许值特殊字符说明
分钟0-59, - * /任务在每小时的第几分钟执行
小时0-23, - * /任务在每天的第几小时执行
日期1-31, - * / ? L W任务在每月的第几天执行
月份1-12 或 JAN-DEC, - * /任务在每年的第几月执行
星期0-7 或 SUN-SAT, - * / ? L #任务在每周的星期几执行（0=周日）

特殊字符说明

  • * (星号) — 表示"每"，匹配该字段的所有值。如 * * * * * 表示每分钟执行。
  • , (逗号) — 列举多个值。如 0 9,18 * * * 表示每天9点和18点执行。
  • - (连字符) — 表示范围。如 0 9-17 * * * 表示每天9点到17点每小时执行。
  • / (斜杠) — 表示步进。如 */15 * * * * 表示每15分钟执行一次。
  • ? (问号) — 仅用于"日期"和"星期"字段，表示不指定。
  • L — 表示"最后"，如 0 0 L * * 表示每月最后一天。
  • W — 表示最近的工作日。
  • # — 表示第几个星期几，如 0 0 * * 1#2 表示每月第2个周一。

Cron在主流平台中的应用

  • Linux Crontab：使用 crontab -e 编辑定时任务，格式为 * * * * * command
  • GitHub Actions：使用 schedule 事件，支持标准Cron语法
  • Kubernetes CronJob：使用 schedule 字段定义Pod定时运行
  • Jenkins： "Build periodically" 使用Cron表达式触发构建
  • AWS Lambda / CloudWatch Events：使用Cron表达式定义触发规则
  • Spring Boot @Scheduled：支持Cron、fixedDelay、fixedRate三种模式

常见问题 (FAQ)

Q: Cron表达式中的0和7在星期字段有什么含义？
A: 0和7都代表周日(Sunday)。通常使用0，但部分系统（如Quartz）用1-7代替0-6。建议统一使用0表示周日。

Q: 如何设置"每N分钟/小时"执行？
A: 使用 */N 语法。例如 */10 * * * * 表示每10分钟执行； 0 */2 * * * 表示每2小时的整点执行。

Q: Cron表达式可以精确到秒吗？
A: 标准Unix Cron不支持秒级精度。如需秒级调度，可使用Quartz Scheduler（7字段格式 秒 分 时 日 月 星期 年）或systemd timer。
