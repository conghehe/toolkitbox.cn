📚 SSH 密钥详解

SSH（Secure Shell）密钥对是现代服务器管理和代码托管的核心安全机制。与密码认证不同，SSH 密钥使用非对称加密技术，由公钥（Public Key）和私钥（Private Key）组成。公钥部署在服务器上，私钥保存在本地设备，两者配合完成身份验证，无需在网络上传输密码。

支持的算法对比

算法安全等级性能兼容性推荐场景
ED25519 极高 极快 OpenSSH 6.5+ 现代系统首选
ECDSA P-256 高 快 广泛支持 通用场景
RSA 4096 高 较慢 所有系统 最大兼容性
RSA 2048 中 中等 所有系统 基本使用

使用场景

  • GitHub / GitLab：将公钥添加到账户设置中，实现免密 git push/pull
  • 服务器管理：将公钥写入服务器的 ~/.ssh/authorized_keys，实现免密 SSH 登录
  • CI/CD 管道：部署密钥用于自动化构建和部署流程
  • SFTP 文件传输：安全的文件传输认证

安全提示

  • 私钥绝不分享：私钥等同于你的数字身份，泄露后任何人都可以冒充你
  • 使用密码保护：可以用 ssh-keygen -p 为私钥添加密码短语（passphrase）
  • 定期轮换：建议每 6-12 个月更换一次 SSH 密钥
  • 不同用途不同密钥：为个人项目和工作项目使用不同的密钥对
  • 本工具不收集数据：所有密钥在浏览器本地生成，不会上传到任何服务器

常见问题

Q: 如何将公钥添加到服务器？

A: 将生成的公钥内容追加到服务器的 ~/.ssh/authorized_keys 文件中。可以使用命令：echo "公钥内容" >> ~/.ssh/authorized_keys

Q: 为什么推荐 ED25519？

A: ED25519 在相同安全级别下密钥更短（256位 vs RSA 2048位），生成和验证速度更快，且不易受侧信道攻击。自 OpenSSH 6.5 起广泛支持。

Q: 本工具生成的密钥与 ssh-keygen 兼容吗？

A: 是的。本工具使用标准的 PEM（Privacy Enhanced Mail）格式输出，与 OpenSSH ssh-keygen 生成的格式完全兼容。
